Project

Soil Moisture Sensor Web Interface

Real-time soil monitoring system with ESP32 and a custom dark-themed web dashboard. IoT fundamentals applied to agriculture.

Overview

The Problem

Farmers and gardeners often struggle to know when to water their plants. Too much water wastes resources and can damage roots. Too little water stresses plants and reduces yield. Manual checking is time-consuming and imprecise. I wanted to build a system that could automatically monitor soil moisture and provide real-time feedback.

The Solution

An ESP32-based soil moisture monitoring system that reads sensor data and displays it on a custom web dashboard. The dashboard shows real-time moisture levels, historical trends, and alerts when soil is too dry or too wet. The dark-themed UI is easy on the eyes and provides clear, actionable information.

Technical Specifications

Hardware & Components

Microcontroller: ESP32 with WiFi connectivity
Sensor: Capacitive Soil Moisture Sensor (analog output)
Power: USB-powered with option for battery backup
Data Storage: EEPROM for local data logging
Communication: WiFi for real-time data transmission to web server

Key Features

Real-Time Data Display: Live moisture readings updated every few seconds on the web dashboard.
Historical Trends: Graph showing moisture levels over time (hourly, daily, weekly).
Smart Alerts: Notifications when soil is too dry (needs watering) or too wet (risk of root rot).
Calibration Tools: Easy calibration for different soil types and sensor variations.
Dark-Themed UI: Custom web interface with consistent styling and easy readability.
Mobile Responsive: Works on phones, tablets, and desktops.
Development

How It Was Built

This project started as a way to understand IoT data collection and visualization. I began by learning how to read analog sensor data on the ESP32, then moved on to setting up a local web server to display that data. The final step was designing a UI that made the data easy to understand at a glance.

Tech Stack

Backend
ESP32 running C++ code. Reads sensor data, processes it, and serves it via HTTP.
Frontend
HTML/CSS/JavaScript. Custom dark theme. Chart.js for data visualization.
Data Storage
EEPROM on ESP32 for local logging. Optional cloud integration for long-term storage.
Challenges

What I Learned

Sensor Calibration
Capacitive sensors vary between units and soil types. Solution: Implemented a two-point calibration system (dry and wet reference points).
WiFi Stability
WiFi connection could drop, causing data loss. Solution: Implemented local data logging and automatic reconnection logic.
Power Consumption
Continuous WiFi and sensor readings drain power quickly. Solution: Implemented sleep modes and optimized update intervals.
Data Visualization
Displaying trends clearly without overwhelming the user. Solution: Used Chart.js with configurable time ranges and zoom functionality.
