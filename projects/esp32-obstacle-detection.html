Project

ESP32 Obstacle Detection System

Navigation assistant for visually impaired users. Built at IIIT Delhi Electronics Design Carnival, July 2025.

Overview

The Problem

Visually impaired individuals face significant challenges in navigating unfamiliar environments. While guide dogs and canes provide some assistance, they have limitations. I wanted to build a technology-based solution that could provide real-time, adaptive feedback about obstacles in the user's path.

The Solution

An ESP32-based system using an HC-SR04 ultrasonic sensor to detect obstacles. The system provides proximity-adaptive audio feedback through a buzzer that increases in frequency as obstacles get closer. The system also integrates with Blynk for remote control and features a real-time web server with a dark-themed UI for sighted users to monitor the system.

Technical Specifications

Hardware & Components

Microcontroller: ESP32 (dual-core, WiFi-enabled)
Sensor: HC-SR04 Ultrasonic Sensor (range: 2cm - 400cm)
Actuator: Passive Buzzer with PWM frequency control
Display: 16x2 LCD Display (I2C interface)
Connectivity: WiFi (via ESP32) for Blynk integration

Key Features

Proximity-Adaptive Buzzer: Buzzer frequency increases as obstacles get closer, providing intuitive feedback.
Blynk App Integration: Remote mute/control of the system via smartphone app.
Real-Time Web Dashboard: Dark-themed web UI displaying live distance data and system status.
LCD Display: Real-time distance readings on a portable display.
Low Power Consumption: Optimized code for extended battery life.
Development

How It Was Built

This project was developed during the Electronics Design Carnival at IIIT Delhi (July 7-18, 2025). Over two weeks, I attended lectures on circuit design, PCB layout, IoT architecture, and prototyping. The hands-on approach allowed me to move from concept to working prototype quickly.

Key Learning Moments:

Understanding sensor calibration and dealing with real-world noise in ultrasonic readings.
Implementing PWM (Pulse Width Modulation) to control buzzer frequency dynamically.
Building a web server on the ESP32 to serve real-time data to a browser.
Integrating third-party IoT platforms (Blynk) for remote control and monitoring.
Challenges

What I Learned

Sensor Noise & Calibration
Ultrasonic sensors can be noisy. Solution: Implemented a rolling average filter to smooth readings.
Power Management
WiFi on ESP32 drains battery quickly. Solution: Implemented sleep modes and optimized WiFi connection intervals.
Frequency Mapping
Converting distance to buzzer frequency in a way that feels intuitive. Solution: Logarithmic mapping to match human perception.
User Testing
Getting feedback from actual users to refine the system. Solution: Collaborated with accessibility experts during the carnival.
Impact & Future

What's Next?

This project demonstrated the potential of IoT for accessibility. Future enhancements could include:

Multiple Sensors: Add sensors on multiple sides for 360-degree obstacle detection.
Machine Learning: Train a model to distinguish between different types of obstacles (walls, people, stairs).
GPS Integration: Add location tracking and route planning for navigation assistance.
Wearable Form Factor: Miniaturize the system into a wearable device (glasses, armband, etc.).
