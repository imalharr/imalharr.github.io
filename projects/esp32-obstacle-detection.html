cat how-it-works.txt

HC-SR04 fires ultrasonic pulse, measures echo return time
distance = (echo_time × 0.034) / 2 → in cm
rolling average filter applied to smooth noisy readings
distance mapped to PWM frequency via logarithmic scale
log scale matches human pitch perception — feels natural
closer obstacle → higher buzzer frequency → stronger warning
LCD shows live distance reading in real time
ESP32 hosts web server → browser dashboard at local IP
Blynk app → remote mute / system control via smartphone

cat build-notes.txt

The actual hard part
Reading through datasheets and pin diagrams to figure out where exactly everything connects. The HC-SR04 docs aren't written for beginners and the ESP32 has a lot of pins. Got it wrong twice before it worked.
Figuring out the code
Getting the PWM frequency mapping to feel right took iteration. Linear mapping sounded wrong — equal distance steps felt unequal in pitch. Switched to logarithmic, which is how human hearing actually works.
Sensor noise
Raw HC-SR04 readings are jittery. Implemented a rolling average to smooth them — without this the buzzer stuttered and was unusable.
Context
Built this among B.Tech students and engineers at IIIT Delhi EDC. Certificate signed by Prof. A V Subramanyam, HoD ECE. Partners: Keysight, LPKF, Bergen Associates.
